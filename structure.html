Building the Structure

HTML provides structure. Any budding web designer or web developers needs to grasp the concept of the 'big' three. These are the core open standard skills that will enable to build beautiful looking, dynamic web pages as well as mobile applications.

Web designers often talk about 'semantic markup'. Semantics is the study of meaning. The heading tags are great examples of semantic markup as they are tags with meaning. That is a <h1> is heading one â€“ the most important heading in the document.

Lists are a very popular way of presenting information and with the addition of CSS can be styled to produce the likes of menu bars. The HTML for a list requires two elements <ul> (Unordered List) used to define the list itself and <li> (List Item) used to declare each item in the list. The <ul> is the parent of the <li>. That is the <li> tags need to nested inside the <ul>.

Semantic mark-up

It is the role of the HyperText Markup Language HTML to provide structure to your document.

HTML is a markup language made up of elements that are nested inside each other to structure the information held in the document.

Presentation

Cascading Style Sheets CSS represent the presentation layer that prescribe appearance - the look and feel - of the a web page.

A key concept in the use of CSS is the separation of document content from document presentation. The HTML dictates the content, the CSS the presentation.

Interaction

It is the role of Javascript to add interaction to a web page. Javascript can do this by intepreted the structure of the HTML - known as the DOM - and change the HTML and/or CSS used in the document. Changes can be event driven, from user interaction, or programmed into the page, allowing Javascript to be used for all kinds of applications from sliding banner adverts to games.
